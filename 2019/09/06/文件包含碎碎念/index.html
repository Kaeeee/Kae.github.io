文件包含

代码注入的一种，原理是注入一段用户能够控制的脚本或代码，让服务器执行。

老年痴呆的简略提点

  • 无限制访问本地敏感信息，敏感文件目录（略）

常见的文件包含函数如下：

PHP:
include(),include_once(),require(),require_once(),fopen(),readfile()…

JSP/Servlet:
ava.io.file(),java.io.filereader()…

ASP:
include file,include virtual…

PHP文件包含函数：
include(),include_once(),require(),require_once()当使用这四个函数包含一个新文件时，该文件将作为PHP代码执行，php内核并不在意该被包含的文件是什么类型都会当成php代码来执行。
通常利用方法：将恶意代码保存成txt或者图片格式上传，执行包含脚本。

文件包含利用条件：
+ include() 等函数通过动态变量的方式引入需要包含的文件。
+ 用户能控制该动态变量。

远程文件包含：
+ php的配置选项allow_url_include函数为on

本地文件包含漏洞利用技巧：
+ 包含用户上传的文件
+ 包含data://或php://input等伪协议
+ 包含session文件
+ 包含日志文件，比如Web Server 的access Log

产生原因

文件包含函数加载的参数没有靖哥哥过滤或者严格的定义，可以被用户控制，包含其他而已文件，导致了执行了非预期的代码。

远程文件包含

PHP配置文件 allow_url_fopen（是否允许打开远程文件）和allow_url_include（是否允许include\require远程文件）设置为ON

include/require等函数可以加载远程文件，如果远程文件没经根严格的过滤，导致了执行恶意文件的代码，这就是远程文件包含漏洞。

PHP伪协议

PHP带有很多内置URL风格的封装协议，可用于类似fopen(),copy(),file_exists()和filesize()的文件系统函数。除了这些封装协议，还能通过stream_wrapper_rigister()来注册自定义的封装协议。

php:// 输入输出流：允许访问PHP的输入输出流、标准输入输出和错误描述符，内存中、磁盘备份的临时文件流以及可以操作其他读取写入资源的过滤器。

php://filter本地磁盘文件进行读取：只是读取需要开启allow_url_fopen,不需要开启allow_url_include

php://input:可访问原始数据的只读流。可读取POST上没有解析的原始数据。enctype=” multipart/form-data”的时候php://input是无效的。
标志性函数:file_get_contents()

数据利用POST方式传递。

利用php://input写入木马时需要同时开启allow_url_fopen 和 allow_url_include (PHP < 5.3.0),即POST方式传递可执行PHP代码。

命令执行和写木马条件相同，直接在burp中POST语句

file://伪协议访问本地文件系统，读取文件内容。

data://伪协议数据流封装器，将原本的include的文件流重定向到了用户可控制的输入流中,如若allow_url_include=On（PHP < 5.3.0）,就可以造成任意代码执行

phar://伪协议不论后缀是什么，都会被当作压缩包来解压。PHP >= 5.3.0 压缩包需要zip，rar不行。用法：将木马文件压缩后该为其他任意格式的文件。
